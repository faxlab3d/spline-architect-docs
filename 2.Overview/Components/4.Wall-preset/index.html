    1. What is a Wall Preset?¬∂

    A Wall Preset is a Struct that defines how a generated SplineArchitectWall actor will look. It is located in the Preset category of the SplineArchitectWall actor (Details panel).

    2. What do the Wall Preset parameters do?¬∂

    Parameter Example
    WallHeight
    ‚Äì Sets the wall height; the scale is applied to all meshes based on this value.
    HeightOverride
    ‚Äì If true, allows manually setting the wall height; otherwise, WallHeight is taken from the first element of the WallMeshes array.
    WallLength
    ‚Äì Sets the length of a wall segment; segments will be divided by this value, and meshes from the WallMeshes array will be selected based on the segment length.
    LengthOverride
    ‚Äì If true, allows manually setting the wall length; otherwise, WallLength is taken from the first element of the WallMeshes array.
    CornerLength
    ‚Äì Sets the corner length; this much is subtracted from segments to form a corner.
    CornerLengthOverride
    ‚Äì If true, allows manually changing the corner length; otherwise, CornerLength is taken from the CornerMesh.
    RandomizeSegmentsBy
    ‚Äì Randomizes the length of the segments. Can be used to add variety to the wall, as a different mesh can be selected based on the segment length, which adds variation.
    WallZHandling
    ‚Äì Defines how meshes are arranged when spline segments are not flat.

    KeepZ ‚Äì Keeps the "Pitch" rotation at 0 and only adjusts the mesh‚Äôs Z position.
    FollowSpline ‚Äì Rotates the mesh to match the segment‚Äôs "Pitch".
    Skew ‚Äì Generates a new, bent mesh.
    KeepFlat ‚Äì Ignores Z-axis changes and generates a straight wall.


    SkipPieces
    ‚Äì Allows skipping or keeping certain segments (either straight or tilted).
    Inset
    ‚Äì Pushes meshes inwards or outwards from the spline by a specified distance.
    OverlapPiecesBy
    ‚Äì The amount of overlap to prevent gaps between meshes due to imperfect edges.
    NumFloors
    ‚Äì How many times to duplicate segments with a Z offset equal to WallHeight (number of floors).
    TreatSplineReversed
    ‚Äì Allows placing meshes in this SplineArchitectWall in reverse order.
    WallMeshes
    ‚Äì The main meshes for wall segments; the most suitable one is selected from this array based on segment length.
    FillerWallMeshes
    ‚Äì Additional meshes to fill gaps, preventing the main WallMeshes from stretching or shrinking.

    MinFillerWallLength
    ‚Äì The minimum length for a filler mesh; shorter meshes will not be placed.
    KeepWorldScale
    ‚Äì If true, meshes will not be scaled according to the segment, but will use their original scale.
    WorldScale
    ‚Äì The original scale of the meshes, applied when KeepWorldScale is true.
    CornerMesh
    ‚Äì The mesh used for forming corners.
    CornerType
    ‚Äì The type of corner: Straight, Bend, Chamfer, or Pipe. (Pipe will be used for pipes with a centered pivot).
    OverlapCornersBy
    ‚Äì Allows overlapping corners individually to avoid gaps, for example, if the ends of a mesh are sticking out.
    OffsetBy
    ‚Äì Offsets all meshes by the specified vector.
    RandomizeLocationBy
    ‚Äì Randomly changes the position of meshes by this vector (small values are recommended to avoid Z-fighting).
    FloorSurfaces
    ‚Äì An array of floor configurations, where each element describes the parameters of the floor surface.

    PostConfigurations
    ‚Äì An array of post configurations, determining how and where posts will be generated.
    RandomisationSettings
    ‚Äì Randomization settings that allow to change specific meshes into other random meshes.
    MaterialParameters
    ‚Äì An array of material parameters that allows setting specific material parameters on certain meshes.

    FloorSurfaces parameters (FFloorData):¬∂

    Parameter Example
    Material
    ‚Äì The material that will be applied to the floor surface.
    AlignTo
    ‚Äì Floor alignment relative to the main walls: Top, Center, or Bottom.
    ZOffset
    ‚Äì Vertical offset for the floor position.
    Inset
    ‚Äì Insetting the floor inwards or outwards from the wall contour.
    Thickness
    ‚Äì The thickness of the floor (extrude height).
    Subdivisions
    ‚Äì Controls the number of polygons in the floor.
    BendsResolution
    ‚Äì Sets the distance between vertices on curved corners.
    UVScale
    ‚Äì Changes the scale of the floor's UV coordinates.
    FillColor
    ‚Äì Vertex color applied to all vertices except the edges.
    EdgesColor
    ‚Äì Vertex color for the edges.
    ColorBlurIterations
    ‚Äì More iterations will blur the floor mesh's vertex colors more.
    UseDelaunayTriangulation
    ‚Äì Delaunay triangulation provides better triangulation, but edges cannot overlap. If the floor mesh isn't working, try disabling this parameter.

    PostConfigurations parameters (FPostData):¬∂

    Parameter Example
    PostMesh
    ‚Äì The mesh used for generating posts.
    GenerationMethod
    ‚Äì Post generation method: OnCorners, OnSplinePoints, BetweenSegments, OnCustomPieces, Periodically, StartAndEnd.
    AverageDirections
    ‚Äì Only applicable when GenerationMethod is OnSplinePoints. Averages the directions.
    PeriodDistance
    ‚Äì Only applicable when GenerationMethod is Periodically. Sets the distance between posts.
    PostOffset
    ‚Äì Offsets the posts by a specified vector (X - along the spline, Y - inwards/outwards, Z - vertically).
    AddRotation
    ‚Äì Adds rotation to all posts.
    bScaleToWallHeight
    ‚Äì Automatically changes the post's scale to match WallHeight.
    PostScale
    ‚Äì Adjusts the post's scale.
    SkipRule
    ‚Äì Can skip posts if the segment is flat or tilted.

    RandomisationSettings parameters (FRandomSettings):¬∂

    Parameter Example
    RandomMeshes
    ‚Äì An array of FRandomMeshes structs that define the rules for random mesh swapping.

    FRandomMeshes parameters:¬∂

    Parameter Example
    KeyMesh
    ‚Äì The target mesh to be replaced by a random one from the RandomMeshes array (including KeyMesh itself).
    RandomMeshes
    ‚Äì An array of meshes with weights (FRandomMeshWithWeight) from which one is randomly selected. A higher weight increases the chance for that mesh.
    FitRandomType
    ‚Äì MatchScale or KeepScale - determines whether to maintain the original scale or to scale the mesh to fit into the key mesh.
    Knockout
    ‚Äì Controls how many meshes will be randomly removed (0.0f = none, 1.0f = all).
    MirrorX/Y/Z
    ‚Äì Option to multiply the scale by -1. Only works when converting to StaticMeshComponents.
    Rot90
    ‚Äì Randomly rotates meshes by 90, 180, or 270 degrees.
    Rot180
    ‚Äì Randomly rotates meshes by 180 degrees only.

    MaterialParameters parameters (FMatParams):¬∂

    Parameter Example
    MaterialApplicationMode
    ‚Äì Material application mode: Include (only specified), Exclude (except specified), All (for all).
    MaterialReferences
    ‚Äì A list of materials to apply parameters to (when the mode is not All).

    ‚Äì Mesh selection mode: Include, Exclude, or All. We can change Material parameters only on specific meshes if needed.
    StaticMeshReferences
    ‚Äì A list of meshes to apply parameters to (when the mode is not All). We can change Material parameters only on specific meshes if needed.
    Parameters
    ‚Äì An array of material parameters (FMatParam).

    ParameterType

    FloatValue

    ColorValue
