    Wall Preset¶

    1. What is a Wall Preset?¶

    A Wall Preset is a Struct that defines how a generated SplineArchitectWall actor will look. It's located in the Preset category of a SplineArchitectWall actor (in the Details panel).

    2. What do the Wall Preset parameters do?¶

    FloorSurfaces parameters (FWallPreset):¶

    Parameter Example
    WallHeight
    – Sets the height of the wall; this value determines the scale for all meshes.
    HeightOverride
    – If true, allows manual override of wall height; otherwise, WallHeight is taken from the first WallMeshes array element.
    WallLength
    – Sets the length of the wall; spline segments will be divided according to this value.
    LengthOverride
    – If true, allows manual override of wall length; otherwise, WallLength is taken from the first WallMeshes array element.
    CornerLength
    – Sets the corner length; this value is subtracted from segments to form a corner.
    CornerLengthOverride
    – If true, allows manual override of corner length; otherwise, it's taken from CornerMesh.
    RandomizeSegmentsBy
    – Randomly alters segment lengths up to the given threshold.
    WallZHandling
    – Determines how to handle Z-axis height differences between segments (KeepZ, FollowSpline, Skew).
    SkipPieces
    – Allows skipping specific segments (straight or angled).
    Inset
    – Offsets meshes inward or outward from the spline by a given distance.
    OverlapPiecesBy
    – Overlap size to prevent gaps caused by imperfect mesh edges.
    NumFloors
    – Number of times to duplicate the segments vertically with offset equal to WallHeight (number of floors).
    WallMeshes
    – Main meshes for wall segments; the best fit is chosen based on segment length.
    FillerWallMeshes
    – Additional meshes used to fill gaps so that the main WallMeshes don't stretch or shrink.
    MinFillerWallLength
    – Minimum length for filler meshes; shorter meshes won’t be used.
    KeepWorldScale
    – If true, meshes are not scaled to fit the segment, and retain their original scale.
    WorldScale
    – Original scale applied to meshes when KeepWorldScale is true.
    CornerMesh
    – Mesh used to form corners.
    CornerType
    – Type of corner: Straight, Bend, Chamfer, or Pipe.
    OffsetBy
    – Offsets all meshes by a given vector.
    RandomizeLocationBy
    – Randomizes mesh positions within the given vector (recommended small values to avoid Z-fighting).
    FloorSurfaces
    – Array of floor surface configurations; each element defines the parameters of a floor surface.
    PostConfigurations
    – Array of post configurations determining how and where posts will be generated.
    RandomisationSettings
    – Randomization settings for mesh distribution and transforms.
    MaterialParameters
    – Array of material parameters allowing customization of specific material properties on certain meshes.

    FloorSurfaces parameters (FFloorData):¶

    Parameter Example
    Material
    – Material applied to the floor surface.
    AlignTo
    – Floor alignment relative to wall: Top, Center, or Bottom.
    ZOffset
    – Vertical offset for floor placement.
    Inset
    – Inward or outward offset of floor from the wall contour.
    Thickness
    – Floor thickness (extrusion height).
    Subdivisions
    – Controls polygon count of the floor.
    BendsResolution
    – Vertex spacing for curved corners.
    UVScale
    – Scales UV coordinates of the floor.
    FillColor
    – Vertex color applied to all non-edge vertices.
    EdgesColor
    – Vertex color for edges.
    ColorBlurIterations
    – More iterations = smoother vertex color blending.
    UseDelaunayTriangulation
    – Enables Delaunay triangulation for better mesh quality. If the floor mesh doesn't work, try disabling this.

    PostConfigurations parameters (FPostData):¶

    Parameter Example
    PostMesh
    – Mesh used for generating posts.
    GenerationMethod
    – Method for placing posts: OnCorners, OnSplinePoints, BetweenSegments, OnCustomPieces, Periodically, StartAndEnd.
    AverageDirections
    – Used only when GenerationMethod is OnSplinePoints. Averages directions.
    PeriodDistance
    – Used when GenerationMethod is Periodically. Defines distance between posts.
    PostOffset
    – Offsets posts by a vector (X = along spline, Y = inward/outward, Z = vertical).
    AddRotation
    – Adds rotation to all posts.
    bScaleToWallHeight
    – Automatically scales post to match WallHeight.
    PostScale
    – Adjusts the scale of the post.
    SkipRule
    – Can skip posts if the segment is flat or angled.

    RandomisationSettings parameters (FRandomSettings):¶

    Parameter Example
    RandomMeshes
    – Array of FRandomMeshes structs defining random replacement rules for meshes.

    FRandomMeshes parameters:¶

    Parameter Example
    KeyMesh
    – Target mesh to be randomly replaced by one from the RandomMeshes array (including itself).
    RandomMeshes
    – Array of meshes with weights (FRandomMeshWithWeight), from which a random one is selected. Higher weight = higher chance.
    FitRandomType
    – MatchScale or KeepScale – whether to scale the mesh to fit or keep its original size.
    Knockout
    – Controls how many meshes to randomly remove (0.0f = none, 1.0f = all).
    MirrorX/Y/Z
    – Allows scaling by -1 on specific axes. Only works when converting to StaticMeshComponents.
    Rot90
    – Randomly rotates meshes by 90, 180, or 270 degrees.
    Rot180
    – Randomly rotates meshes by 180 degrees only.

    MaterialParameters parameters (FMatParams):¶

    Parameter Example
    MaterialApplicationMode
    – Application mode: Include (only listed), Exclude (all except listed), or All.
    MaterialReferences
    – List of materials to apply parameters to (when mode is not All).
    MeshApplicationMode
    – Mesh selection mode: Include, Exclude, or All. Lets you change material parameters on specific meshes.
    StaticMeshReferences
    – List of meshes to apply parameters to (when mode is not All).
    Parameters
    – Array of material parameters (FMatParam).

    FMatParam parameters:¶

    Parameter Example
    ParameterName
    – Name of the material parameter to modify.
    ParameterType
    – Type of parameter: Float or Vector.
    FloatValue
    – Value when type is Float.
    ColorValue
    – Value when type is Vector (color).
